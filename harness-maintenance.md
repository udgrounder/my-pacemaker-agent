# 하네스 시스템 유지보수 가이드

> **대상:** 하네스 프로젝트 자체를 관리하는 사람  
> **이 파일은 프로젝트에 설치되지 않는다** (harness 루트에만 존재)

---

## 업그레이드 후보 반영

프로젝트들에서 수집된 `.agents-workspace/upgrade-candidates/` 파일을 하네스 템플릿에 반영한다.

### 원칙

- **사용자 요청으로 시작한다.** 에이전트가 자발적으로 하네스 파일을 수정하지 않는다.
- **후보 하나씩 진행한다.** 전체 목록을 먼저 제시하고 사용자가 선택한 항목을 하나씩 처리한다.
- **일괄 자동 적용 금지.** 하네스 파일 변경은 모든 프로젝트에 전파되므로 사용자가 내용을 검토하면서 진행한다.

### 작업 절차

1. `.agents-workspace/upgrade-candidates/` 파일 목록을 읽고 전체를 사용자에게 제시한다:

```
## 하네스 개선 후보 목록

| 파일 | 제목 | 발견 상황 | 적용 범위 |
|------|------|---------|---------|
| [파일명] | [제목] | [발견 상황 요약] | 이 프로젝트 / 모든 프로젝트 |

→ 반영할 항목을 선택해주세요.
```

2. 사용자가 선택한 항목을 **하나씩** 다음 형식으로 보여준다:

```
## 후보: [제목]
발견 상황: [내용]
적용 범위: [이 프로젝트 / 모든 프로젝트]

현재 방식:
[현재 어떻게 되어 있는지]

개선 방안:
[변경할 내용]

적용 대상 파일:
- [파일 경로]

→ 반영할까요? (예 / 아니오 / 수정 후 반영)
```

3. 사용자 응답 후 다음 후보로 넘어간다. 모든 선택 항목이 처리될 때까지 반복한다.
4. 반영 완료된 파일을 `.agents-workspace/upgrade-candidates/` 에서 제거한다.

---

## 완료 기준

- [ ] 후보 전체 목록 사용자에게 제시됨
- [ ] 선택된 항목이 모두 사용자 확인 후 처리됨
- [ ] 처리 완료된 upgrade-candidates 파일 제거됨
