# Layer 0 — 하네스 업데이트 / 재설치 세션

> **사용 시점:** 기존 프로젝트에 하네스를 업데이트하거나 재설치할 때

---

## 역할

`.agents-workspace/personas/architect.md` 를 읽고 그 역할로 작업한다.

---

## 목적

이 세션은 두 가지 방향의 흐름을 처리한다:

```
프로젝트 → 하네스 (아웃바운드)
  이 프로젝트에서 발견한 개선 후보를 하네스 프로젝트로 이전

하네스 → 프로젝트 (인바운드)
  하네스 템플릿의 최신 내용을 이 프로젝트에 반영
```

---

## 작업 순서

### 1단계 — 아웃바운드: upgrade-candidates 수집

`.agents-workspace/upgrade-candidates/` 를 확인한다.

파일이 있는 경우:
1. 각 파일을 읽고 목록으로 정리한다
2. 사용자에게 보고한다:

```
## 이 프로젝트에서 발견된 하네스 개선 후보

| 파일 | 제목 | 발견 상황 | 적용 범위 |
|------|------|---------|---------|
| [파일명] | [제목] | [발견 상황 요약] | 이 프로젝트 / 모든 프로젝트 |

→ 하네스 프로젝트로 이전할 항목을 선택해주세요.
```

3. 사용자가 선택한 항목을:
   - **하네스 프로젝트 경로를 알 수 있는 경우**: 해당 경로의 `.agents-workspace/upgrade-candidates/` 로 복사
   - **경로를 모르는 경우**: 이전할 파일 목록과 내용을 요약해 사용자에게 전달 (수동 이전)

4. 이전 완료된 항목은 `.agents-workspace/upgrade-candidates/` 에서 제거한다.

파일이 없는 경우: 이 단계를 건너뛴다.

---

### 2단계 — 인바운드: 하네스 템플릿 업데이트 반영

> 실제 파일 교체는 `install.py --upgrade` 로 수행한다.  
> 이 단계는 업데이트 후 프로젝트 상태를 확인하는 절차다.

사용자에게 확인한다:
- 하네스 버전 업데이트 후 변경된 파일 목록
- 이 프로젝트에서 커스터마이즈된 내용이 덮어써진 항목 없는지

> **주의:** 커스터마이즈된 내용이 덮어써졌다면 git diff로 확인 후 복원한다.

---

### 3단계 — project_memory 정합성 확인

업데이트 후 `workspace/project_memory/` 파일이 현재 프로젝트 상태와 맞는지 간단히 확인한다:
- `shared/architecture.md`: 아키텍처 결정 이력이 최신인가?
- `shared/contracts.md`: 인터페이스 계약이 현재 코드와 맞는가?
