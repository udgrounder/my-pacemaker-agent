# 시스템 비평 전 실제 상태 미검증 패턴
**타입**: 방법론 개선
**발견 상황**: 이번 세션 — dist/ 동기화 이슈 비평 시 dist_sync.py 훅이 이미 작동 중임을 확인하지 않고 "diverged 상태"로 단정함

## 현재 방식
시스템 비평 시 문서·코드 분석만으로 "이 부분이 문제다"고 판단한다.
실제 구현 상태(훅 설정, 스크립트 존재 여부 등)를 먼저 확인하지 않아 이미 해결된 문제를 새 문제로 오진한다.

## 개선 방안
시스템 비평 항목을 도출할 때:
1. 문제 제기 전 실제 파일·설정을 먼저 확인한다
2. "X가 없다"고 주장하기 전에 X가 실제로 없는지 grep/read로 검증한다
3. 비평 결과에 "검증 여부"를 명시한다 (검증됨 / 추정)
