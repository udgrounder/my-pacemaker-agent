# 스킬: 계획서 인터뷰

> 사용자가 plan.md를 검토할 때 도움을 요청하면 실행한다.  
> 기존 스킬을 순서대로 활용하고, 결과를 plan.md에 반영하는 오케스트레이터다.

---

## 목적

사용자가 결정해야 할 것을 함께 채워나가며 계획서를 완성한다.  
비평(plan_critic)과 다르다 — 약점을 찾는 게 아니라 **결정 공백을 채우는 것**이다.

---

## 실행 순서

### 1단계 — 명시적 결정 항목 처리

plan.md의 `에이전트 보고 > 사용자 결정 필요` 항목을 하나씩 사용자와 논의한다.

→ 에이전트의 자연스러운 대화 능력을 사용한다.  
→ 각 항목에 대해 선택지와 영향을 설명하고, 사용자의 결정을 받아 plan.md에 즉시 반영한다.

항목이 없으면 이 단계를 건너뛴다.

---

### 2단계 — 숨은 결정 발굴

`.mpa-workspace/skills/analysis/silent_decision_extraction.md`의 프롬프트 템플릿을 plan.md에 적용한다.

→ 추출된 숨은 결정 포인트를 사용자에게 하나씩 제시하고, 방향을 함께 결정한다.  
→ 결정된 내용은 plan.md의 `예상 조용한 결정` 섹션에 추가한다.

발견된 항목이 없으면 이 단계를 건너뛴다.

---

### 3단계 — 실패 시나리오 확인

`.mpa-workspace/skills/analysis/counterexample_finding.md`의 계획서 반례 탐색 템플릿을 plan.md에 적용한다.

→ 발견된 시나리오를 사용자에게 제시하고, 대응 방향을 결정한다.  
→ 결정 내용은 plan.md의 `반례` 섹션에 추가한다.

발견된 시나리오가 없으면 이 단계를 건너뛴다.

---

## plan.md 업데이트 규칙

| 결과 | 반영 위치 |
|------|---------|
| 결정된 사용자 결정 필요 항목 | 에이전트 보고 섹션 — 체크 후 결정 내용 기록 |
| 발굴된 숨은 결정 + 방향 | `예상 조용한 결정` 섹션에 추가 |
| 실패 시나리오 대응 | `반례` 섹션에 추가 |
| 미결정 항목 | `미결정 — 구현 시 확인` 으로 기록 |

---

## 종료

모든 단계 완료 또는 사용자가 중단 요청 시:

변경 요약(결정된 항목 수, 미결정 항목)을 짧게 보고하고 승인 여부를 묻는다.
