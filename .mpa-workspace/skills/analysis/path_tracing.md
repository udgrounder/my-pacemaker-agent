# 스킬: 경로 추적

> 코드의 정상/실패 경로를 하나씩 끝까지 따라가며 논리 검증하는 기법  
> 사용 시점: 구현 결과 확인, 통합 체크포인트

---

## 언제 쓰는가

- 구현된 코드를 검토할 때
- "코드가 돌아가는 것"과 "의도대로 동작하는 것"을 구분해야 할 때
- 코드 전체를 읽는 것이 비효율적일 때 (전체 읽기 대신 핵심 경로만 깊이)

---

## 핵심 원리

코드 전체를 읽으려 하면 오히려 논리 오류를 놓친다.  
하나의 경로를 입력부터 출력까지 끝까지 따라가는 것이 효과적이다.

```
정상 경로 1개   →  의도한 흐름이 맞는가
실패 경로 1개   →  에러 발생 시 의도한 대로 처리되는가
```

---

## 프롬프트 템플릿

### 정상 경로 추적

```
아래 코드에서 [구체적인 정상 시나리오]가 실행될 때의
경로를 입력부터 출력까지 순서대로 추적해줘.

각 단계에서:
- 어떤 값이 전달되는가
- 어떤 결정이 내려지는가
- 의도한 동작과 일치하는가

[코드 붙여넣기]
```

### 실패 경로 추적

```
아래 코드에서 [구체적인 실패 시나리오: 네트워크 오류/DB 실패/입력값 오류 등]가
발생했을 때의 경로를 추적해줘.

확인 항목:
- 에러가 어디서 발생하는가
- 에러가 어떻게 전파되는가
- 최종적으로 어떻게 처리되는가
- 사용자에게 어떤 응답이 돌아가는가

[코드 붙여넣기]
```

---

## 출력 활용 방법

경로 추적 결과를 태스크 계획의 **검증 체크리스트**와 대조한다.

```
- [ ] 정상 경로: [시나리오] — 추적 완료, 의도와 일치
- [ ] 실패 경로: [시나리오] — 추적 완료, 에러 처리 확인
```
