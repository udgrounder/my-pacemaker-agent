# 스킬: 조용한 결정 추출

> Agent가 구현 중 임의로 결정해버릴 수 있는 지점을 미리 찾아내는 기법  
> 사용 시점: 설계 검토, 계획서 검토

---

## 언제 쓰는가

- 계획서나 요구사항이 완성된 후, 구현 요청 전
- "이 계획대로 구현하면 agent가 알아서 결정해버릴 것 같다"는 느낌이 들 때
- 결과물에서 "왜 이렇게 만들었어?"가 반복될 때

---

## 핵심 원리

계획서에 명시되지 않은 부분 = agent가 조용히 채우는 부분

```
명시된 것        →  agent가 따름
명시되지 않은 것  →  agent가 임의로 결정 → 의도 간격 발생
```

---

## 프롬프트 템플릿

```
아래 계획서(또는 요구사항)를 보고, 구현하다가 내가 예상 못 한
복잡도나 결정이 필요한 지점을 모두 찾아줘.

조건:
- 계획서에 명시되어 있어도 해석의 여지가 있는 것 포함
- 각 항목마다 "agent가 선택할 수 있는 방향들"을 함께 제시할 것
- "이렇게 하면 되겠지"라고 넘어갈 수 있는 모든 지점을 찾을 것

[계획서 내용 붙여넣기]
```

---

## 출력 활용 방법

추출된 항목을 태스크 계획의 **"예상 조용한 결정"** 섹션에 옮기고, 각 항목마다 원하는 방향을 명시한다.

```
| 결정 포인트          | 원하는 방향        |
|---------------------|-------------------|
| (추출된 항목 1)      | (내가 원하는 방향) |
```
