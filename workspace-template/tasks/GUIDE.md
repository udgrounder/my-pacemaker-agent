# Tasks 레이어 가이드

> **목적:** 작업 단위로 계획서와 내역서를 관리하고, 진행 상태를 추적한다.

---

## 구조

```
workspace/tasks/
├── INDEX.md              ← 전체 작업 색인
├── active/               ← 진행 중인 작업
│   └── [작업명]/
│       ├── plan.md       ← 계획서 (요청 원문 + 설계)
│       └── changelog.md  ← 작업 내역서 (구현 완료 후 작성)
└── done/                 ← 완료된 작업
    └── [작업명]/
```

---

## 작업 라이프사이클

```
작업 생성 → active/[작업명]/plan.md 작성 → 사용자 검토
         → 작업 진행 → changelog.md 작성 → 사용자 검토
         → 완료 → done/[작업명]/으로 이동
```

---

## 파일 역할

| 파일 | 작성 시점 | 용도 |
|------|---------|------|
| `plan.md` | 작업 시작 전 | 요청 원문 + 계획. 사용자 검토 후 승인 |
| `changelog.md` | 작업 완료 후 | 변경 파일/메소드/내역. 검증 agent가 plan.md와 비교 |

---

## 작업명 규칙

간결하고 내용을 반영하는 이름을 사용한다.
```
예) user-auth-bugfix, payment-api-refactor, add-search-feature
```

---

## INDEX.md 관리

| 컬럼 | 내용 |
|------|------|
| 작업명 | 폴더명과 동일 |
| 상태 | active / done |
| 요약 | 한 줄 설명 |
| 날짜 | 생성일 |
