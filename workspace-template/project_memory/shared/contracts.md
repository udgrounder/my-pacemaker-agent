# 인터페이스 계약

> **Tier 2** — 변경 시 UI/Backend 양쪽 모두 영향.  
> **Interface-First 원칙:** 코드 구현 전 이 문서 먼저 업데이트.  
> UI 세션 + Backend 세션 모두에 주입됨.

---

## ⚠️ 계약 변경 프로토콜

1. 변경 필요 확인
2. **이 문서 먼저 업데이트** (코드 변경 전)
3. Backend 구현 → UI 구현 순서 (또는 동시 진행 시 계약 문서 기준)
