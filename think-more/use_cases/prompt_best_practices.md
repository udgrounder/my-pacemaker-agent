# Prompt 작성 Best Practices

> **핵심 요점**: LLM에게 **정확하고 명확한 문맥**을 전달하면, 모델이 **올바른 방향**으로 높은 확률로 답변을 생성합니다. 아래는 효과적인 프롬프트를 만들기 위한 가이드와 템플릿, 실전 체크리스트를 정리한 문서입니다.

---

## 1️⃣ 핵심 원칙

| 원칙 | 실전 팁 |
|------|--------|
| **구체성** | “파일을 정렬해 줘” → “`src/utils` 디렉터리 안에 있는 `.js` 파일을 알파벳 순서로 정렬해 주세요.” |
| **제한된 범위** | 질문을 **작은 단위**(한 파일, 한 함수, 한 개념)로 나눔 |
| **배경 제공** | 현재 상황·제약을 **간단히** 서술 (예: “프로젝트는 Node 12, 의존성은 `express`만 있습니다”) |
| **목표 명시** | 최종 목표를 명확히 제시 (예: “테스트 커버리지를 90 % 이상으로 올리고 싶습니다”) |
| **예시 포함** | 기대 출력 형식·예시를 함께 제공 (예: “JSON 배열 형태로 반환해 주세요”) |
| **키워드 강조** | 중요한 용어·파일명을 **백틱**(```)으로 감싸서 강조 |
| **톤·제한** | 출력 길이·포맷 제한을 명시 (“최대 200 토큰, 마크다운 표 형식”) |

---

## 2️⃣ 구조화된 프롬프트 템플릿

```
[배경]
- 프로젝트: <간단 설명> (예: Node.js, Python Flask)
- 현재 상황: <문제·제약>

[목표]
- 정확히 무엇을 달성하고 싶은가? (예: 특정 함수 구현, 파일 정렬, 성능 최적화)

[제한·요구사항]
- 언어·버전, 의존성, 파일 경로 등
- 출력 포맷·길이 제한

[예시]
```json
{ "result": "…" }
```

[질문]
<핵심 질문>
```

> **예시**
>
> ```
> [배경]
> - 프로젝트: Node.js 14, `express` 기반 API 서버
> - 현재 상황: `src/routes/user.js`에 사용자 인증 로직이 없음
>
> [목표]
> - JWT 기반 로그인 미들웨어를 구현하고 싶습니다
>
> [제한·요구사항]
> - ES6 모듈 사용, `jsonwebtoken` 패키지만 허용
> - 파일 경로: `src/middleware/auth.js`
> - 반환값: Express 미들웨어 형태
>
> [예시]
> ```js
> export const authMiddleware = (req, res, next) => { … };
> ```
>
> [질문]
> 위 요구사항을 만족하는 `authMiddleware` 코드를 작성해 주세요.
> ```

---

## 3️⃣ 실전 팁 (LLM 활용 시)

1. **단계별 요청**
   - 1차: “요구사항을 요약해 주세요.”
   - 2차: “위 요약을 바탕으로 코드 초안을 만들어 주세요.”
2. **검증·피드백**
   - LLM이 만든 코드를 **테스트**하거나 **정적 분석**(ESLint 등)으로 검증하고, 결과를 다시 LLM에 알려 “버그를 고쳐 주세요”.
3. **캐시 활용**
   - 동일 파일·구조가 반복될 경우 **해시‑캐시**(앞서 설계)로 파일 내용 재전송을 피하십시오.
4. **비용 절감**
   - 프롬프트에 **길이 제한**을 명시하고, 필요 시 `summarize` 툴로 요약 후 전달 → 토큰 사용량 감소.

---

## 4️⃣ 체크리스트 (프롬프트 작성 전 확인)

- [ ] **배경**을 한두 문장으로 간결히 제시했는가?
- [ ] **목표**가 구체적이고 단일한가?
- [ ] **제한·요구사항**을 모두 열거했는가?
- [ ] **예시**(출력 포맷·예시 코드)를 포함했는가?
- [ ] **길이·형식 제한**을 명시했는가?

---

## 5️⃣ 결론

- **정확·명확한 문맥** → LLM이 올바른 방향으로 높은 확률로 답변.
- **구조화된 프롬프트**와 **제한·예시** 제공이 핵심.
- 필요 시 **단계별 요청**·**검증 피드백** 루프를 활용해 품질을 높이고 **비용**을 최적화합니다.
