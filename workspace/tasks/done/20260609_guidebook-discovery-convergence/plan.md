## 목적

A에서 4.0이 "의도 보정 나선"과 "발산 처리(새 태스크/증류)"를 선언했으나, 발산이 실제 일어나는 **6.3 구현 후 발견**에는 수렴/발산 구분이 없다. 6.3을 4.0과 정합시켜, "구현 후 발견은 좋은 것"이라는 격려와 "끝없는 재작업(발산)은 분리해야 한다"는 경계를 함께 세운다. 3장 상위 통합 원칙에는 4.0 목적과의 연결 문장을 더한다.

---

## 요구사항

- 6.3: 구현 후 발견이 **수렴(의도가 또렷해짐)**인지 **발산(방향 없는 재작업)**인지 구분하는 관점 추가
- 발산 처리: 사용자 의도가 진짜 바뀜 → 새 태스크 분리(기존 분류와 연결) / agent 진동 → 증류(direction.md)
- 수렴 신호: "이번에 무엇이 또렷해졌는가"를 발견 맥락 기록(4단계)과 연결
- 3장: "구조 설계 = 보정 나선 설계"임을 잇는 연결 1문장 (4.0 참조)
- 기존 격려 톤·절차는 보존, 더하기만 함
