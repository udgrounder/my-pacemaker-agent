## 항목별 변경 상세

### 비평 1 — 라우팅 판단 불가 시 처리
**현재:**
```
판단 불가 시:
  1. 새 기능/버그/리팩터링 → 계획서 작성 후 구현
  2. 작업 결과 검토 → 기존 코드 분석
  3. 간단한 질문 → 바로 답변
```
**변경:**
```
판단 불가 시: 에이전트가 가장 가능성 높은 유형으로 먼저 추론하고,
"[유형]으로 처리하겠습니다. 다르게 처리하시려면 말씀해주세요."로 단축.
```

### 비평 4 — Task 규모 판단 기준
**현재:** "단일 파일 · 명확한 범위 · 비가역적 변경 없음"
**변경:** "영향 범위가 해당 파일로 국한됨 (다른 모듈 인터페이스·전역 설정 변경 없음) · 비가역적 변경 없음"

### 비평 5 — 비평 임계값
**현재:** "기준 1개 이상 해당 시 → 독립 비평 권장"
**변경:**
```
고위험 기준 (1개 이상): 비가역적 변경 / 외부 시스템 연동
중위험 기준 (2개 이상): 가치 결정 미해소 / 타 도메인 영향 / 실패 비용 높음
→ 고위험 1개 OR 중위험 2개 이상 해당 시 → 독립 비평 권장
```

### 비평 7 — 자가 개선 필터
**현재:** 놀라움 필터 + 1개 기준 ("다음에 이 역할이 몰랐다면 실수했을 내용인가?")
**변경:** 자가 개선 필터 + 사실 기반 구체적 조건 3개 추가
```
다음 중 하나 이상 해당 시 기록한다:
- plan.md "에이전트 가정"이 구현 중 틀린 것으로 밝혀졌는가?
- 구현 중 plan.md 자체를 수정해야 했는가?
- 범위 밖 변경을 발견해 분리 보고했는가?
해당 없으면 생략. "다음에 이 역할이 몰랐다면 실수했을 것인가?"는 보조 기준으로 유지.
```

### 비평 8 — Layer 2 자동 제안
**위치:** 작업 완료 섹션 마지막
**추가:**
```
4. **Layer 2 주기 확인** — `workspace/tasks/INDEX.md`에서 마지막 Layer 2 이후
   완료된 태스크 수를 확인한다. 3개 이상이면:
   → "Layer 2 체크포인트 시점입니다. 진행할까요?"
```
