## 변경 상세

### `layer1_design.md` — plan 형식 내 가치 결정 판별 기준

현재 plan 형식:
```
### 사전 결정 필요 사항
> 구현 전에 내가 결정해야 할 것들 (비가시적 위임 후보)
- [ ] 질문 1 — [기술적 결정 / **가치 결정**]

> ⚠️ **가치 결정**("무엇이 더 중요한가")은 AI에게 위임 불가 — 반드시 사용자가 결정
```

변경 후: "가치 결정"이라고 표기하는 기준을 판별 질문으로 제시

**판별 질문 (2개):**
1. "이 결정이 잘못됐을 때 코드 수정만으로 되돌릴 수 있는가?" → Yes → 기술 결정
2. "이 결정이 비즈니스 목표·우선순위·사용자 경험에 따라 달라지는가?" → Yes → 가치 결정

**가치 결정 패턴 (예시 목록):**
- 어떤 기능을 포함/제외할 것인가 (범위 결정)
- 트레이드오프에서 어느 쪽을 택할 것인가 (성능 vs 일관성, 단순성 vs 기능성)
- 오류 발생 시 사용자에게 어떻게 보여줄 것인가 (UX 방향)
- 외부 정책·법규·비즈니스 규칙에 따른 선택

**기술 결정 패턴 (예시 목록):**
- 동일 기능의 구현 방법 선택 (라이브러리, 알고리즘)
- 코드 구조·패턴 선택 (기능 동등성이 확보된 경우)
- 성능 최적화 접근법
