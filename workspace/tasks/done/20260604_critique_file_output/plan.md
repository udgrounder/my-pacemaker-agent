# 태스크: 독립 비평 파일 출력
**상태:** 구현 완료  
**목적:** 서브에이전트 비평 결과를 메인 에이전트를 거치지 않고 critique.md에 직접 저장, completion bias 차단

---

## 에이전트 보고

**사용자 결정 필요:** 없음

**에이전트 가정:**
| 가정 | 근거 | 틀렸을 때 영향 |
|------|------|--------------|
| 서브에이전트가 Write 도구로 파일을 직접 저장할 수 있다 | Claude Code Agent 도구는 파일 쓰기 포함 전체 도구 접근 가능 | 저장 실패 시 결과 유실 — 완화: 저장 후 경로 반환으로 확인 |
| 메인 에이전트가 "저장 완료" 메시지만 받으면 critique 내용에 접근 못 한다 | Agent 도구 반환값 = 서브에이전트의 최종 텍스트 응답 | 메인 에이전트가 파일을 직접 읽으면 여전히 볼 수 있음 — 완화: 메인 에이전트 지시에 "critique.md를 읽지 말라" 명시 |

## 핵심 변경 내용

### `layer1_critique.md` — 서브에이전트 프롬프트 템플릿 수정

현재: 산출물을 채팅 텍스트로 반환  
변경: `workspace/tasks/active/[태스크명]/critique.md`에 직접 저장 + "저장 완료: [경로]"만 반환

추가할 지시:
```
## 저장 지시 (필수)
비평 결과를 채팅 응답으로 출력하지 않는다.
Write 도구로 `workspace/tasks/active/[태스크명]/critique.md`에 직접 저장한다.
저장 완료 후 반환값: "critique.md 저장 완료: [경로]"
```

### `agent_rules.md` — "비평 필요 여부 자동 평가" 섹션 수정

현재: 서브에이전트 결과를 메인 에이전트가 받아 사용자에게 제시  
변경: 메인 에이전트는 파일 경로만 안내, critique.md 내용을 읽거나 요약하지 않는다

```
비평 완료 후 메인 에이전트 안내:
"비평 결과가 [경로]에 저장됐습니다. 직접 확인 후 구현 진행 여부를 알려주세요."
```

## 반례

- **시나리오 1**: 서브에이전트 파일 저장 실패 → 반환값에 경로가 없어 메인 에이전트가 감지 가능. 완화: 저장 성공 여부를 반환값으로 확인하도록 지시
- **시나리오 2**: 메인 에이전트가 "사용자에게 내용을 전달해야 한다"는 판단으로 critique.md를 직접 읽음 → "읽지 말라" 명시 지시로 완화
