## 목적

가이드북의 **체계 목적 진술**을 재정립한다. 사용자는 분명한 목적을 갖고 요청하지만, 그것을 전혀 다른 존재인 agent에게 그대로 전달하기 어렵다 — **목적(핵심)은 사전에 존재하나, 구체적 형태는 결과물을 매개로 한 보정의 반복으로 수렴한다.** 체계는 고정 명세를 명중시키는 도구가 아니라 이 **전달·보정의 나선**을 효율적으로 돌리는 도구다. 현재 4.1의 "엔트로피/수렴/예측가능" 프레이밍이 체계를 "출력 통제 장치"로 오해하게 만드는 문제를 교정하고, 빈칸 처리 규율(**가시화 + 영향도 분기**)을 목적 층위에서 명시한다.

---

## 요구사항

토론에서 합의된 목적 진술 v3를 문서에 반영한다. v3의 핵심 골격:

```
대전제:    사용자 의도에 부합하는 결과물 (긍정형)
의도의 성격: 목적(핵심)은 사전에 존재 / 구체적 형태는 흐릿 → 전달이 어려움
전달 문제:  LLM은 깊은 지식 보유 / 단 맥락 약한 빈칸은 일반값으로 채움
            → 특수 의도일수록 더 많은 전달 필요, 부족하면 어긋남
보정의 정체: 맥락을 더해 LLM 조건부 분포를 의도 쪽으로 좁히는 것
수렴 방식:  결과물을 매개로 한 차이 확인 + 재전달의 반복 (선형 아닌 나선)
            — 사람/agent 무관, 편차는 대상별 크기 차이
빈칸 규율:  채움 자체는 문제 아님 / 문제는 ①가시화 부재 ②영향도 미판단
            → 영향 작으면 채우고 검토요청, 크면 멈추고 질문
나선 3요소: ① 보정을 앞당김(빠른 결과물) ② 발견을 잃지 않음(증류) ③ 수렴 포착
제약:      준수 지속가능성 — 절차가 부담 한계 넘으면 폐기 → minor 경로
발산 처리:  사용자 발산 = 새 태스크 / agent 진동 = 증류
```

부수 합의: 검수·발견 장치(가이드북의 ③ 계열) 강조는 "방어 편향"이 아니라 의도 보정이 본질이기에 **정당하다** — 이번 A 범위에서는 목적 진술이 이 점과 모순되지 않게만 쓰고, 2부 무게중심 재조정은 B/C로 미룬다.
