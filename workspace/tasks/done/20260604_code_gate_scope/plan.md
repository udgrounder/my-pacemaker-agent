# 태스크: 코드 게이트 태스크 범위 강화

**목적:** `code_gate.py`가 "어떤 태스크든 하나라도 승인됐으면 모든 소스 수정 허용"하는 문제를 수정한다. 승인된 태스크의 범위 안에서만 소스 수정을 허용한다.

**상태:** 설계 완료

---

## 에이전트 보고

**사용자 결정 필요:**

| 항목 | 선택지 | 영향 |
|------|--------|------|
| 범위 이탈 시 동작 | **A. 경고(warn)** — 차단 없이 컨텍스트 주입 | 기존 작업 흐름 유지, 범위 이탈이 눈에 띔 |
| | B. 차단(block) | 실수로 허용 범위 밖 수정 시 작업 중단 위험 |
| 빈 `.approved` 처리 | **A. 기존 동작 유지** (빈 파일 = 모든 경로 허용) | 단순 작업에서 경로 작성 부담 없음, 하위 호환 |
| | B. 항상 경로 명시 필수 | 강제력 높아지나 기존 워크플로우 전면 변경 필요 |

> **에이전트 권장:** 범위 이탈 → 경고(A), 빈 `.approved` → 기존 유지(A). 이유: 게이트가 너무 강하면 정상 작업이 막혀 MPA 시스템 자체가 거부감을 만든다.

**에이전트 가정:**

| 가정 | 근거 | 틀렸을 때 영향 |
|------|------|--------------|
| `.approved` 파일 내용은 현재 모두 비어 있음 | 기존 생성 규칙 확인 | 경로 파싱 로직 조정 필요 |
| `dist/` 동기화 필요 | code_gate.py가 dist/에도 존재 | dist/ 반영 누락 |

|------|---------|

| `.mpa-workspace/core/agent_rules.md` | "작업 진행" `.approved` 생성 단계에 경로 작성 안내 추가 |
| `dist/.mpa-workspace/hooks/code_gate.py` | 동기화 |
| `dist/.mpa-workspace/core/agent_rules.md` | 동기화 |

## `.approved` 파일 형식 (신규)

```
# 허용 경로 (한 줄에 하나, 접두사 매칭)
src/features/auth/
tests/auth/
```

- **비어 있으면** → 기존과 동일하게 모든 소스 수정 허용 (하위 호환)
- **경로가 있으면** → 해당 접두사에 해당하는 경로만 허용, 이탈 시 경고

## 구현 단계

- [ ] 1. `code_gate.py` — `has_approved_task()` → `get_approved_scopes()` 로 변경: `.approved` 내용을 읽어 경로 목록 반환
- [ ] 2. `code_gate.py` — `main()` 에 범위 검사 로직 추가: 경로 목록이 있으면 대상 파일이 해당 범위인지 확인, 이탈 시 `emit_warn`
- [ ] 3. `agent_rules.md` — "작업 진행" `.approved` 생성 단계에 안내 추가: 실패 비용 Level 1 태스크는 plan.md의 수정 대상 파일 목록을 `.approved`에 함께 작성
- [ ] 4. `dist/` 동기화

## 검증 체크리스트

- [ ] 빈 `.approved` → 기존과 동일하게 동작
- [ ] 경로 명시 `.approved` → 범위 내 파일 수정 허용
- [ ] 경로 명시 `.approved` → 범위 외 파일 수정 시 경고 주입
- [ ] `.approved` 없음 → 차단 (기존과 동일)

## 반례

- 에이전트가 plan.md 수정 대상 파일을 빠뜨리고 `.approved`에 작성 → 정상 파일이 경고 대상이 됨 → 경고(차단 아님)이므로 작업은 계속됨
- `.mpa-workspace/` 파일 수정 → `is_always_allowed()`로 게이트 통과, 범위 검사 대상 아님 (기존과 동일)
