## 변경 상세

### Zone 정의 (agent_rules.md 상단 또는 작업 생성 섹션 위에 추가)

```
## 사용자 개입 등급

| Zone | 강도 | 적용 조건 | 처리 방식 |
|------|------|---------|---------|
| Zone 1 | 하드 게이트 | 소스 수정 전 / 실패 비용 Level 1 구현 시작 전 | 기계적 차단 또는 명시적 확인 없이 진행 불가 |
| Zone 2 | 강조 확인 | 가치 결정 항목 / 🚨 리뷰 항목 / major 완료 후 Layer 2 | 핵심만 꺼내서 짧게 확인 요청 |
| Zone 3 | 자동 처리 | 문서 업데이트 / 상태 기록 / 낮은 위험 판단 | 실행 후 한 줄 고지 또는 무음 처리 |
```

### 계획서 제시 변경 (agent_rules.md "작업 생성" + layer1_design.md)

현재:
```
계획서를 검토해 주세요. (검토는 필수입니다)
· 직접 검토 / · 인터뷰 지원
```

변경 후:
```
계획서가 준비됐습니다. ([파일 경로])

확인이 필요한 항목:
[가치 결정 항목이 있으면]
  ❓ 가치 결정: [항목] — [선택 A] / [선택 B]
[에이전트 가정 중 "계획 변경" 영향인 것이 있으면]
  ⚠️ 검증 필요 가정: [내용]

→ 위 항목에 답하시면 구현을 시작합니다.
  계획 전체를 검토하시려면 파일을 열어주세요.
  인터뷰 지원이 필요하면 "인터뷰 해줘"라고 말씀해주세요.
```

가치 결정 항목도, 높은 영향 가정도 없으면:
```
계획서 준비 완료. ([파일 경로]) — 바로 구현할까요?
```

### 검토 리포트 응답 요구 변경 (layer1_review.md)

현재: 리포트 제시 후 사용자 응답 대기 (단계 구분 없음)

변경 후:
- 🚨 즉시 수정 항목이 있으면 → 응답 필수, 수정 진행 여부 확인
- ⚠️ 항목만 있으면 → "위 주의 항목을 인지하셨으면 계속 진행합니다" (응답 없어도 진행 가능)
- ✅만 있으면 → 바로 다음 단계 안내 (응답 요구 없음)

### 세션 종료 자동 처리 (layer1_implement.md)

"세션 종료 시" 항목 1~9를 실행 후 변경된 항목만 간략히 출력:
```
세션 종료 처리 완료
  · changelog.md — [주요 변경 요약 1줄]
  · architecture.md — [추가/수정된 결정 요약] (변경 없으면 생략)
  · contracts.md — [변경 내용 요약] (변경 없으면 생략)
  · INDEX.md 상태 갱신
```
"다음 작업 예측"만 남기고 나머지는 사용자에게 승인 요구하지 않는다.

### Layer 2 트리거 변경 (agent_rules.md "작업 완료")

현재: major 1개 OR minor 5개 → 동일한 강조 메시지

변경 후:
- major 완료 → Zone 2 제안 (현재와 동일한 메시지)
- minor 5개 → 한 줄 약하게 제안: `💡 minor 태스크 5개 누적 — 전체 정합성 점검(Layer 2)을 고려해보세요.`
