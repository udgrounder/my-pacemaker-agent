## 변경 내용 상세

### 변경 1 — minor 판단 기준 강화 (agent_rules.md + layer1_design.md)

**현재:**
> 변경이 한 파일 또는 단일 관심사에 국한된다
> 구현 전 설계 결정이 필요 없다 (방법이 자명함)
> git reset으로 즉시 복구된다

**변경 후:**
조건마다 "불확실하면 No"를 명시:
> - 변경이 한 파일 또는 단일 관심사에 국한된다 **— 코드를 열어보지 않아 불확실하면 No**
> - 구현 전 설계 결정이 필요 없다 **— 방법이 자명하지 않으면 No**
> - git reset으로 즉시 복구된다 **— 외부 상태(DB·파일시스템·외부 API) 변경이 있으면 No**

### 변경 2 — minor 계획서 검토 요청 (agent_rules.md)

**현재 minor 흐름:**
```
최소 plan.md 작성 → 즉시 approve → 구현
```

**변경 후:**
```
최소 plan.md 작성 → 사용자에게 계획서 제시 + 검토 요청 → approve → 구현
```

검토 요청 형식:
```
minor 태스크: [목적]. 계획대로 진행합니다.
[계획서 핵심 1-2줄]
→ 다른 방향이면 말씀해 주세요. 없으면 바로 시작합니다.
```

- major와 달리 명시적 승인 불필요 — 거부나 방향 변경 요청이 없으면 진행
- 사용자가 응답하면: 피드백 반영 후 진행
- 사용자가 응답 없으면: 바로 구현 시작
