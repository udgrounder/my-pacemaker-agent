# 태스크: 독립 비평 기술적 격리

**목적:** `layer1_critique.md`의 비평 독립성을 "새 스레드 권고"에서 "서브에이전트 실행"으로 격상한다. 선언으로 컨텍스트를 분리하는 것은 실제로 completion bias를 막지 못하므로, 기술적으로 격리된 서브에이전트가 기본 경로가 된다.

**상태:** 설계 완료

**파생 출처:** MPA 시스템 비평 인터뷰 — 모순 2 (독립 비평 독립성 문제)

---

## 에이전트 보고

**사용자 결정 필요:** 없음

**에이전트 가정:**

| 가정 | 근거 | 틀렸을 때 영향 |
|------|------|--------------|
| Claude Code 환경에서 Agent 도구(서브에이전트)를 사용 가능 | 이번 세션에서 실제로 사용함 | fallback 경로만 남게 됨 |
| 다른 환경(codex, gemini)에서는 서브에이전트 미지원 가능 | 도구 목록 차이 | fallback(새 스레드) 경로 유지 필요 |

## 변경 상세

### `layer1_critique.md` — 스레드 선택 섹션 변경

**현재:**
```
| 새 스레드를 열 수 있는 경우 | 새 스레드에서 시작한다 |
| 새 스레드를 열 수 없는 경우 | 같은 스레드에서 선언 후 진행 |
```

**변경 후:**
```
| Claude Code 환경 (Agent 도구 사용 가능) | 서브에이전트로 실행한다 — 기본 경로 |
| 기타 환경 (Agent 도구 미지원) | 새 스레드에서 시작한다 |
| 새 스레드도 불가한 경우 | 같은 스레드 선언 방식 — 독립성 미보장, 명시적 고지 필요 |
```

서브에이전트 실행 시 전달할 것:
- `workspace/memory/shared/project_identity.md`
- `workspace/memory/shared/architecture.md`
- `workspace/memory/shared/contracts.md` (존재 시)
- 비평 대상 `plan.md` 경로
- `personas/plan_critic.md` 내용 (또는 경로)

전달하지 않을 것:
- 설계 세션 대화 내용 (컨텍스트 격리 핵심)

### `agent_rules.md` — "비평 필요 여부 자동 평가" 섹션 변경

**현재:**
```
→ 새 스레드에서 독립 비평을 진행하면 완성도가 높아집니다.
```

**변경 후:**
```
→ 서브에이전트로 독립 비평을 진행하면 완성도가 높아집니다. (Agent 도구 미지원 환경: 새 스레드)
```
