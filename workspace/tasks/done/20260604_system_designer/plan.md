|------|------|--------------|

## 신규 페르소나: 체계 설계자 (System Designer)

**핵심 특성 — 일반 Task Designer와 다른 점:**

| 항목 | Task Designer | System Designer |
|------|-------------|-----------------|
| 수정 대상 | 소스 코드 | `.mpa-workspace/` 규칙 파일 |
| 영향 범위 | 이번 태스크 | 모든 미래 세션 전체 |
| 비평 방식 | 새 스레드 필수 | 사용자 결정 A/B |
| 코드 게이트 | 적용 | 미적용 |
| 핵심 관심사 | 구현 정확성 | 규칙 간 일관성 |

**역할 규칙 (신규):**
- 변경 전: 영향 받는 다른 규칙 파일을 모두 읽는다
- 변경 후: 수정된 파일과 참조하는 파일 간 불일치를 확인한다
- 새 용어 도입 시: 기존 파일 전체에서 동일 개념을 가리키는 다른 용어가 있는지 확인한다
- 규칙 삭제 시: 해당 규칙을 참조하는 다른 파일이 있는지 확인한다
