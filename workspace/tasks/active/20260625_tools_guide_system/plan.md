## 목적

README가 구조적으로 빠뜨리는 **운영자 관점 정보**를 일정한 틀로 정리하는 도구 가이드 체계를 `tools/`에 만든다. 첫 사례로 graphify·serena를 채워 틀을 검증한다.

---

## 요구사항

1. **정체성**: README가 끝나는 곳에서 시작하는 운영자 관점 문서. 도구의 *일반 운영 성질*을 담음(내 환경 상태 스냅샷 아님).
2. **2단 구조**:
   - 머릿말(멘탈 모델): 언제 필요한가(한 줄 결론) → 사상(태그) → 동작 방식(운영 메커니즘만, 간단히)
   - 본문(운영 절차): ②설치 ③연결 ④프로젝트별 ⑤팀공유 ⑥유지보수 ⑦비용·주의·보안 ⑧되돌리기 ⑨산출물위치 ⑩생명주기
3. **채움 원칙**: 검증 가능한 출처(공식 docs·소스·신뢰 사례) 기반. 직접 설치 불필요. 불확실/상충 항목은 "미확인" 표시.
4. **위치**: repo 최상위 `tools/` (guidebook과 형제).
5. **차별화**: 동작 방식·기능 설명은 README 복제가 아니라 운영에 영향 주는 메커니즘만.
