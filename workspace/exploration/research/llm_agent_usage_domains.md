# LLM 에이전트 주요 사용 도메인 — 웹 스캔 (2026-06-16)

> 목적: MPA를 "일반 코어 + 작업 도메인 팩"으로 분리·확장할 가치가 있는지, 어느 도메인부터 지원할지를 *실제 사용 분포*로 검증.
> 방법: 가벼운 웹 스캔(3 검색 + 1 fetch). 깊은 검증 아님 — 윤곽 잡기용.
> 필터: 단순 사용량이 아니라 **"에이전트적(다단계) × 중대·되돌리기비용 있음"** = MPA 방법론이 값을 하는 작업.

---

## 검증 대상 (사용자 가설)

사용자 예상 상위 3: ① 프로그램 개발 ② 회사를 위한 자료 조사 ③ 깊은 이해를 위한 주제 토론.

---

## 스캔 결과

### 개인/지식노동 사용 (Brookings 전국 설문)
- **인터넷 검색/정보 탐색 74%** — 가장 흔한 개인 용도.
- **문서 작성/편집** — 직업 활동 최상위(학사 이상 35%).
- 데이터 분석, 고객 소통 뒤따름.
- 개인 목적 생성형 AI 사용 57%.

### 엔터프라이즈 에이전트 사용
- **고객 지원이 배포 1위** (Klarna AI가 채팅 2/3 처리, 응답 11분→2분, 2025 ~$6천만 절감).
- 영업 리서치·아웃리치, 문서 처리, **R&D 리서치 에이전트**(Genentech gRED Research Agent), 지식 관리, 공급망, 사이버보안.
- 65% 기업이 일부 워크플로우를 에이전트로 자동화, 2026 +33% 성장 전망. Gartner: 2026까지 엔터프라이즈 시스템 40%가 작업별 에이전트 탑재(전년 <5%).

## 가설 보정

| 도메인 | 사용량 | 에이전트적 × 중대 (MPA 적합) | 판정 |
|--|--|--|--|
| **개발(코딩)** | 상위 | 높음 (다단계·중대·가역비용) | ✅ 확정 — 상위 + 적합 |
| **자료 조사(리서치)** | **최상위** (정보탐색 74%, R&D 에이전트) | 높음 (탐색→검증→종합) | ✅ 확정 — 상위 + 적합. 가장 강한 갭(MPA 미보유) |
| **주제 토론** | 중간 (개인 지식노동) | 중간 (다회 교환이나 게이트·산출물 약함 = agentic-lite) | △ 실재하나 가벼움. 데이터상 "상위 라벨"은 아님 |
| 문서/콘텐츠 작성 | **상위** (작성 35%) | 중~상 (다단계 문서 생산) | ➕ 가설이 과소평가. 후보로 추가 검토 |
| 고객 지원 | **사용량 1위** | **낮음** (건당 저중대·고볼륨 자동화, 게이트 무의미) | ❌ 방법론 필터 탈락 — 사용량 1위지만 MPA 부적합 |
| 데이터 분석 | 상위 | 중~상 | ➕ 후보 |

---

## 결론

1. **사용자 가설 ①②는 강하게 확인** — 개발·리서치는 상위 사용 + MPA 방법론 적합. 특히 **리서치가 최상위 사용인데 MPA엔 작업 도메인으로 없음(deep-research 스킬만 존재) = 가장 명확한 확장 갭**.
2. **③ 토론은 실재하나 "agentic-lite"** — MPA discussion_mode가 게이트를 벗긴 게 데이터와 일치. 상위 사용 도메인이라기보다 개인 지식노동 보조.
3. **가설이 놓친 후보 = 문서/콘텐츠 작성** — 사용량 상위 + 다단계라 MPA 적합 가능. 리서치의 *출력*과 겹치기도 함(자료조사→보고서).
4. **"사용량 1위 = 적합"은 거짓** — 고객 지원이 사용량 1위지만 건당 저중대·고볼륨이라 MPA 방법론이 값을 못 함. **사용량이 아니라 "중대·다단계"로 골라야** 한다는 우리 필터가 데이터로 확인됨.

→ **확장 우선순위(적합도 기준):** 개발(보유) → 리서치(최우선 갭) → 문서작성(검토) → 토론(보유, 경량 유지).
